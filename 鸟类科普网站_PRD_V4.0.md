## 1. 项目概述

### 1.1 项目背景

本项目为**四专题模块化鸟类科普自适应网站**，延续原有科普网站清新自然视觉风格，剔除冗余板块，固定四大核心科普场景。网站主打**双浏览体验模式**，分别适配目的性检索用户、沉浸式探索用户，支持 PC / 手机双端自适应。全站采用浅层级架构、组件化复用开发，搭配轻量化趣味交互，解决传统科普网站枯燥、查找繁琐、适配性差的问题，满足不同用户的科普学习、问题检索、公益参与需求。

**与旧版图鉴网站的关系**：旧版鸟类图鉴网站（鸟类地球 Online）废弃，本站为全新独立网站，不做具体鸟类种类百科介绍，专注四大科普专题。旧版网站中的鸟类趣味转盘交互代码予以沿用，作为首页探索模块的基础。

### 1.2 核心用途

专注鸟类垂直科普，核心覆盖宠物鸟饲养护理、文明观鸟规范、野生鸟类救助、鸟类保护四大场景。支持图文、插画、视频、趣味交互等多元内容展示，内置全站搜索检索功能，满足用户精准查资料、趣味学知识两类核心需求。所有文章页面采用可复用组件开发，支持持续更新内容，迭代维护成本低。

### 1.3 核心用户

- 观鸟爱好者、观鸟入门新手
- 想要学习鸟类保护、野生鸟类救助知识的用户
- 新手养鸟用户、遇到养鸟突发问题的资深养鸟用户
- 想要参与鸟类公益组织、鸟类调研活动的用户

### 1.4 核心使用场景

- 目的性用户：快速检索观鸟、养鸟、野鸟救助、鸟类保护相关专项知识，解决实际问题
- 探索性用户：碎片化沉浸式浏览，趣味了解鸟类知识，拓展科普认知
- 户外偶遇受伤野鸟，需要标准化紧急救助指导
- 新手养鸟入门、解决饲养过程中疾病、护理等突发问题
- 学习文明观鸟规范，规避伤鸟违规行为，参与鸟类公益保护活动

## 4. 全局通用布局规范（双端统一）

### 4.1 顶部常驻导航栏

首页、分类列表页固定顶部常驻导航，沉浸式文章阅读页隐藏该导航，保障阅读体验。

**常驻栏组成（从左至右）**：网站 Logo + 四大专题快捷入口（养鸟日常 / 观鸟指南 / 鸟类救助 / 鸟类保护）+ 全站搜索栏

#### 4.1.1 搜索栏核心功能

- 支持实时联想检索，输入文字自动弹窗展示**相关专题入口、相关文章入口**
- 支持模糊匹配：标题、标签、简介关键词均可检索
- 回车或点击联想条目，直接跳转对应专题页 / 文章页

#### 4.1.2 双端适配规则

- **PC 端**：横向完整展示 Logo、四大分类、搜索栏，布局居中均衡
- **手机端**：Logo 居左，搜索栏居右，四大分类收纳为汉堡菜单，点击展开

### 4.2 全站双浏览体验定位

网站针对两类用户做**差异化体验分流**，首屏直接让用户选择浏览模式，精准匹配需求：

- **模式一：目的性检索模式**（适配熟悉鸟类、有明确需求、快速找资料的用户）
- **模式二：沉浸式探索模式**（适配新手、无明确目标、休闲学习、趣味探索的用户）

## 5. 首页完整布局与交互规范

首页支持鼠标滚轮上下滑动，整体分为**首屏选择区、精准检索内容区、趣味探索内容区**三大板块，逻辑连贯、体验分层。

### 5.1 首页第一板块：首屏 Hero 选择区

核心作用：引导用户选择浏览方式，完成两类用户体验分流，全屏背景插画，氛围感拉满。

#### 5.1.1 页面布局（PC 端左右分栏）

- **右侧：双无框功能卡片（核心交互入口，权重差异化）**
  - 卡片 1（主入口 / 高优）：插画 + 文字【进入专题】——对应**目的性快速检索模式**，服务刚需、问题解决型用户，视觉层级优先、默认主推
  - 卡片 2（辅入口 / 次优）：插画 + 文字【随便看看】——对应**沉浸式趣味探索模式**，服务休闲、新手学习型用户，视觉层级次之

- **左侧：网站信息介绍区**
  - 顶部：网站主标题（自定义网站名称）
  - 中部：网站简短介绍文案
  - 底部：引导问句——「您想以什么方式探索网站？」

#### 5.1.2 移动端适配

上下流式布局：上方网站名称 + 介绍，下方双卡片横向并列展示，适配竖屏浏览。

### 5.2 首页第二板块：目的性检索内容区

面向有明确需求的用户，支持快速预览、切换、进入四大专题，交互联动、实时联动更新。

#### 5.2.1 布局结构（左右自适应可变布局）

- **左侧：可变信息介绍区**
  - 固定模块：网站交互介绍、专题标题、一句话简介、【快速进入专题】按钮
  - 核心特性：跟随右侧卡片 hover 状态**实时可变更新内容**

- **右侧：四大专题无框插画卡片**
  - 四张无边框卡片，分别对应：养鸟日常、观鸟指南、鸟类救助、鸟类保护
  - 每张卡片搭配专属主题插画，简洁无边框

#### 5.2.2 核心联动交互规则

面向休闲探索用户，轻量化趣味交互，弱化目的性，主打趣味科普、随机探索。

#### 5.3.1 分屏布局（左右二分屏）

- **左侧：今日鸟种盲盒模块**：随机展示鸟类科普卡片，每日更新，支持点击查看详情

  - 转盘顶部新增引导文字介绍
  - 转盘上方添加 easing 动画对话气泡：「我是什么鸟？」
  - 点击气泡 / 转盘对应区域，弹出匹配的鸟类科普文章卡片
  - 后置交互需求（低优先级）：鼠标悬浮转盘对应位置，自动播放对应鸟类音频

## 8. 文章内容模块类型清单（V3.0 新增）

文章正文区由以下 6 种标准模块自由组合而成，模块顺序、数量不限，均通过 Markdown 语法书写，Astro 构建时自动渲染为对应组件样式。所有模块样式由全局组件统一控制，**禁止单篇文章定制样式**。

### 8.1 文字模块

用途：正文科普段落、知识点说明、注释补充。

Markdown 写法：普通段落正文，支持加粗高亮关键词（`**关键词**`），支持行内主色标注。

渲染规则：正文字号、行高、段间距全局统一，重点词汇以主色品牌绿高亮显示，不使用粗黑刺眼样式。

### 8.2 图片模块

用途：原创插画、实景配图、场景示意图。

Markdown 写法：标准 `![图片说明](图片路径)` 语法，图片文件统一存放于 `/public/images/` 目录下对应专题子文件夹。

渲染规则：图片统一圆角裁切展示，自适应容器宽度，无直角硬边，无拉伸变形，支持点击放大预览（Lightbox）。

### 8.3 视频模块

用途：科普讲解视频、案例演示视频。

Markdown 写法：使用自定义 shortcode 语法（Astro 组件调用）：

```
<VideoEmbed url="B站iframe地址" title="视频标题说明" />
```

渲染规则：16:9 固定比例自适应布局，铺满父容器，手机 / PC 端无变形、不横向溢出，支持全屏播放，样式与页面整体风格统一。

### 8.4 引用块模块

用途：法律法规原文引用、权威机构说明、来源标注。

Markdown 写法：标准 `>` 引用语法。

渲染规则：左侧主色竖线标注，背景浅色填充，字体颜色稍弱于正文，与普通段落视觉区分清晰。

### 8.5 步骤列表模块

用途：救助操作步骤、饲养流程指引、观鸟行为规范等有序 / 无序操作说明。

Markdown 写法：
- 有序步骤：标准 `1. 2. 3.` 有序列表
- 无序要点：标准 `- ` 无序列表

渲染规则：有序列表以步骤序号圆圈标注，主色填充；无序列表以主色小圆点标注；行间距宽松，适配长文本阅读，禁止使用默认浏览器样式。

### 8.6 高亮框模块

用途：重要提示、救助警示、禁止行为提醒、科普重点知识框。

Markdown 写法：使用自定义 shortcode 语法（Astro 组件调用），支持三种语义类型：

```
<Callout type="tip">   提示类：绿色，用于建议、推荐做法 </Callout>
<Callout type="warning"> 警示类：黄色，用于注意事项、风险提醒 </Callout>
<Callout type="danger">  危险类：红色，用于禁止行为、违规警示、紧急救助注意事项 </Callout>
```

渲染规则：圆角卡片样式，左侧色条 + 对应功能色背景浅填充，图标 + 文字组合，视觉区分明确但不过于刺眼，与全站低饱和调性保持一致。

## 9. 交互规范（统一落地规则）

### 9.1 首页专题卡片联动交互

- PC 端 hover 高亮机制：当前卡片提亮、其余卡片灰度弱化，视觉聚焦
- 左右区域实时联动，无刷新切换专题信息，交互流畅无卡顿
- 双端点击反馈统一：轻微缩放回弹动效

### 9.2 鸟类转盘趣味交互

- 气泡动画：上方对话气泡「我是什么鸟？」采用柔和 easing 缓动动画，动态悬浮效果
- 点击交互：点击气泡 / 转盘区域，弹出对应鸟类科普文章悬浮卡片
- 音频交互（低优先级迭代）：悬浮触发对应鸟类鸣声音频，自动播放、切换页面自动暂停

### 9.3 文章页沉浸式交互

- 抽屉目录顺滑展开 / 收起，无生硬跳转
- 章节滚动精准定位，高亮跟随阅读位置
- 图片、视频自适应适配，无溢出、无变形

## 10. 组件化开发规范

### 10.1 核心可复用组件清单

所有文章页面统一复用以下组件，通过 Markdown 文件更新内容，无需修改前端代码：

- 文章头部信息组件（标题、简介、标签）
- 抽屉式目录导航组件
- 文字内容组件
- 图片展示组件（含 Lightbox）
- 视频播放组件（Bilibili iframe，16:9 自适应）
- 引用块组件
- 步骤列表组件（有序 / 无序）
- 高亮框组件（tip / warning / danger 三种）
- 底部相关推荐组件
- 首页专题联动卡片组件
- 盲盒 / 转盘趣味交互组件

### 10.2 硬性开发要求

- 所有科普文章零定制页面，全部复用统一模板
- 首页双模式、联动交互、趣味组件全局统一样式与动效
- 严格区分**检索模式**与**探索模式**体验逻辑，互不冲突

## 11. 视觉设计规范（沿用现有 Demo UI 体系）

本网站**完全沿用已开发完成的科普网站 Demo 视觉风格、色彩体系、UI 质感、动效逻辑**，整体保持清新、治愈、轻量化自然科普调性，无全新视觉改版，仅针对本项目四专题架构、双体验模式做适配延伸，保证系列网站视觉统一性。

### 11.1 整体视觉基调

- **设计风格**：清新自然、轻量化极简、低饱和度治愈风，扁平化 + 轻微肌理质感，无厚重阴影、无高强度描边
- **页面气质**：安静、舒适、年轻化、科普感强，弱化严肃说教感，保留专业度同时增加柔和探索氛围
- **整体布局特征**：大量留白、无边框 / 细边框卡片、圆角统一、层次干净、模块分割柔和

### 11.2 色彩系统（沿用 Demo 固定色值）

- **主色调（品牌绿）**：`#84DB5D`，全站按钮、高亮文字、选中状态、主题强调色
- **辅助色调（品牌蓝）**：`#1C9DE4`，搭配渐变、卡片底色、模块区分
- **中性色系统**：纯白背景、浅灰分割、深灰正文、黑色标题
- **功能色**：成功绿（tip）、提示黄（warning）、警示红（danger），用于第 8.6 章高亮框模块

### 11.3 卡片与组件 UI 规范

- **卡片样式统一**：大圆角、低模糊阴影、浅白通透底色，默认无边框 / 极细浅边框

## 14. 技术栈与架构选型（V3.0 新增）

### 14.1 选型结论

**Astro + 本地 Markdown + Pagefind + Tailwind CSS，部署至 Vercel。**

此组合精准命中核心诉求：响应快、全站文章搜索、轻量交互、组件复用、零服务器成本。

### 14.2 各技术层说明

| 层级 | 选型 | 用途 |
|---|---|---|
| 框架 | **Astro** | 页面生成、组件复用、路由、内容集合 |
| 内容存储 | **本地 Markdown 文件** | 文章数据，Git 统一版本管理 |
| 样式框架 | **Tailwind CSS** | 响应式布局，延续 Demo 设计系统 |
| 全站搜索 | **Pagefind** | 构建后自动生成静态搜索索引，零后端 |
| 交互层 | **原生 JS**（少量） | 转盘、专题联动、动效，轻量不引入重型框架 |
| 部署 | **Vercel** | Git 推送自动构建，全球 CDN，免费 |
| 版本管理 | **GitHub** | 代码与文章内容统一管理 |

### 14.3 Astro 框架选型理由

Astro 专为内容型网站设计，构建产物为纯 HTML，无多余运行时 JS 负担，页面加载速度极快。其「内容集合（Content Collections）」功能专为「Markdown 文件 → 自动渲染成页面」场景设计：将 `.md` 文件放入指定目录，构建后自动生成对应文章页面，套用统一组件模板，无需手动维护路由配置。不同专题的分类页、文章页均可使用不同 Astro 模板，组件复用彻底。

### 14.4 Pagefind 搜索选型理由

Pagefind 是专为静态网站设计的本地搜索引擎。Astro 构建完成后，Pagefind 自动扫描全部生成的 HTML 页面，建立搜索索引，索引文件随网站一起部署，**完全不依赖后端服务器**。对本站体量（几十至数百篇文章），搜索响应时间 100ms 以内，满足 PRD 全站标题 / 标签 / 简介关键词模糊匹配需求。

### 14.5 不选其他方案的原因

- **不用纯静态 JSON（旧版方式）**：旧版适合条目式数据（鸟类图鉴），不适合长文本文章。JSON 存储文章内容格式复杂、易出错，全站搜索需自行实现，组件复用靠 JS 字符串拼接，可维护性差。
- **不用 Next.js / Nuxt**：动态应用框架，有运行时服务器，部署复杂，与「纯静态 + 零服务器成本」诉求不符。
- **不用 WordPress / 传统 CMS**：需要 PHP 环境和数据库，部署成本高，响应慢，与轻量化诉求背道而驰。
- **不用云 CMS（Contentful / Notion API）**：内容存于第三方，搜索速度受 API 限速影响，引入外部依赖增加不稳定性。本项目内容由维护者整理后统一上传，不需要可视化编辑后台。

### 14.6 旧版转盘代码迁移方式

旧版网站（鸟类地球 Online）中的转盘核心 JS 逻辑直接迁移至 Astro 项目，封装为独立 Astro 组件（`SpinWheel.astro`）。数据源从原来的硬编码改为从文章 Markdown frontmatter 动态读取（随机抽取若干篇文章标题及 ID 填充转盘分区），不重写动效核心逻辑，仅做数据层适配。
